# Plan 01-01: Install and Configure i18n Library

## Overview

Install react-i18next and related packages, create the i18n configuration file, and integrate it with the React app entry point.

## Why This Matters

This establishes the foundation for all localization work. Without proper i18n infrastructure, we cannot translate any UI text or support language switching.

## Tasks

- [x] **Task 1**: Install i18n dependencies
  - Install: `react-i18next`, `i18next`, `i18next-browser-languagedetector`
  - Command: `pnpm add react-i18next i18next i18next-browser-languagedetector`
  - Files: `package.json`, `pnpm-lock.yaml`

- [x] **Task 2**: Create i18n configuration file
  - Create `src/i18n.ts` with i18next initialization
  - Configure language detector (browser preference)
  - Set fallback language to English
  - Configure namespace to "translation" (default)
  - Enable debug mode for development
  - Files: `src/i18n.ts` (new)

  - Files: `src/locales/en/translation.json`, `src/locales/ko/translation.json` (new)

## Technical Notes

**i18n.ts structure:**
```typescript
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import LanguageDetector from 'i18next-browser-languagedetector';

import enTranslation from './locales/en/translation.json';
import koTranslation from './locales/ko/translation.json';

i18n
  .use(LanguageDetector)
  .use(initReactI18next)
  .init({
    resources: {
      en: { translation: enTranslation },
      ko: { translation: koTranslation }
    },
    fallbackLng: 'en',
    debug: true,
    interpolation: {
      escapeValue: false // React already escapes
    }
  });

export default i18n;
```

**main.tsx integration:**
```typescript
import { Suspense } from 'react';
import { createRoot } from 'react-dom/client';
import './i18n'; // Import i18n config (side-effect)
import App from './app/App.tsx';
import './styles/index.css';

createRoot(document.getElementById('root')!).render(
  <Suspense fallback="Loading...">
    <App />
  </Suspense>
);
```
